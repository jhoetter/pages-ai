# Internationalization

## Locales

- **de**: primary (copy defaults).
- **en**: fallback / dev.

## Rules

- UI uses `i18next` + `react-i18next`; keys: `area.component.field` (e.g. `editor.slash.heading1`).
- CLI: `--locale de|en` or `PAGESAI_LOCALE`; stderr messages localized when human format.

## Locale-aware

- Dates: `Intl.DateTimeFormat`
- Numbers: `Intl.NumberFormat`
- Relative time: `Intl.RelativeTimeFormat`
- File sizes: binary IEC in en, optional de rules

## Block content language

- Optional `lang` on page (`pages.locale` column); editor spellcheck hint.
- Search: Postgres `simple` + `german` configs where available; v1 store `tsvector` with `simple` for all.

## Error codes

Structured: `{ "code": "PAGE_NOT_FOUND", "messageKey": "errors.pageNotFound", "details": {} }` — client maps key to locale.
